# Características de Markdown

Docusaurus admite **[Markdown](https://daringfireball.net/projects/markdown/syntax)** y algunas ** funciones adicionales **.

## Front Matter

Los documentos de Markdown tienen metadatos en la parte superior llamados [Front Matter](https://jekyllrb.com/docs/front-matter/):```text title="my-doc.md"
// highlight-start
---
id: my-doc-id
title: My document title
description: My document description
slug: /my-custom-url
---
// highlight-end

## Markdown heading

Markdown text with [links](./hello.md)
```## Enlaces

Se admiten enlaces regulares de Markdown, utilizando rutas URL o rutas de archivos relativas.```md
Let's see how to [Create a page](/create-a-page).
```

```md
Let's see how to [Create a page](./create-a-page.md).
```**Resultado:** Veamos cómo [crear una página](./create-a-page.md).

## Imágenes

Se admiten imágenes regulares de Markdown.

Puede usar rutas absolutas para hacer referencia a imágenes en el directorio estático (`static/img/your-image.png`):```md
![Your logo](/img/your-image.png)
```<!-- Ejemplo de imagen: añada sus propias imágenes a/img/estático -->

También puede hacer referencia a las imágenes relativas al archivo actual. Esto es particularmente útil para colocar imágenes cerca de los archivos Markdown que las utilizan:```md
![Docusaurus logo](./img/docusaurus.png)
```## Bloques de código

Los bloques de código Markdown son compatibles con el resaltado de sintaxis.````md
```jsx title="src/components/HelloDocusaurus.js"
function HelloDocusaurus() {
return <h1>¡Hola, Docusaurus!</h1>;
}```
````

```jsx title="src/components/HelloDocusaurus.js"
function HelloDocusaurus() {
  return <h1>Hello, Docusaurus!</h1>;
}
```

## Admonitions

Docusaurus has a special syntax to create admonitions and callouts:

```md
:::tip My tip

Use this awesome feature option

:::

:::danger Take care

This action is dangerous

:::
```

:::tip My tip

Use this awesome feature option

:::

:::danger Take care

This action is dangerous

:::

## MDX and React Components

[MDX](https://mdxjs.com/) can make your documentation more **interactive** and allows using any **React components inside Markdown**:

```jsx
export const Highlight = ({children, color}) => (
  <span
    style={{
      backgroundColor: color,
      borderRadius: '20px',
      color: '#fff',
      padding: '10px',
      cursor: 'pointer',
    }}
    onClick={() => {
      alert(`Has hecho clic en el color ${color} con la etiqueta ${children}`)
    }}>
    {children}
  </span>
);

This is <Highlight color="#25c2a0">Docusaurus green</Highlight> !

This is <Highlight color="#1877F2">Facebook blue</Highlight> !
```
